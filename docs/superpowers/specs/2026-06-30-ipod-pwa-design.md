# iPod 风格本地音乐播放器 PWA — 设计文档

- **日期**: 2026-06-30
- **类型**: 全新项目(空目录)
- **目标用户**: 作者本人(单人自用,情绪价值小工具)
- **运行平台**: iPhone(iOS Safari + 添加到主屏幕,全屏 PWA)
- **开发平台**: Windows(纯静态文件,无需 Mac,无构建步骤)

## 1. 目标与背景

做一个跑在 iPhone 上的、还原经典 iPod 体验的本地音乐播放器:用触摸屏模拟经典点按轮盘(Click Wheel),完成切歌、调音量、浏览歌曲等操作。视觉走高级黑 + 玫红点缀的科技高级感路线。

### 已排除的方案及原因
- **遥控网易云 App**: iOS 沙盒隔离,无公开 API 能控制其他 app 的播放、读取其播放状态或歌词。非越狱设备无法实现,**放弃**。
- **原生 iOS App**: 需要 Mac + Xcode + 开发者账号,作者在 Windows 开发,**放弃**。
- **结论**: 做成 **PWA**,播放用户**手动导入并持久保存**的本地音乐文件。

## 2. 范围

### 做(MVP)
- PWA 全屏运行(添加到主屏幕后无浏览器 UI)
- 经典点按轮盘:画圈滚动 + 中心键 + 上/下/左/右四方向键
- 手动导入音频文件,持久保存到 IndexedDB(目标容量:几十首)
- 三个屏幕:主菜单 / 歌曲列表 / 正在播放
- 播放控制:播放、暂停、上一首、下一首、进度跳转
- 应用内音量(Web Audio 增益,非系统音量)
- 「正在播放」界面:每首歌一个**专属固定撞色渐变背景** + 歌名 + 歌手
- 暗夜哑光黑 + 克制玫红点睛的视觉
- 滚动格触感震动(默认开);咔哒音效(设置开关,默认关)
- 离线可用(service worker 缓存应用外壳)

### 不做(留待以后)
- 歌词(留位,不实现)
- 封面自动抓取
- 歌单 / 专辑 / 分组
- 循环 / 随机播放模式(MVP 播完即停)
- 均衡器、评分等

## 3. 技术选型

**纯原生 Web**:HTML + CSS + 原生 JS(ES modules),无框架、零依赖、零构建。
理由:核心价值在定制的轮盘交互与定制视觉,框架帮不上忙反而碍事;播放、存储用浏览器原生 API 即可;在 Windows 上用浏览器即可直接开发调试。

## 4. 架构与文件结构

```
ipod/
├── index.html              # 唯一页面,iPod 外壳骨架
├── manifest.json           # PWA 配置(图标、全屏 standalone、主题色)
├── service-worker.js       # 离线缓存应用外壳
├── icons/                  # 主屏幕图标(多尺寸)
├── styles/
│   └── ipod.css            # 全部视觉:外壳、屏幕、轮盘、黑+玫红主题
└── js/
    ├── app.js              # 启动 + 总协调,串起各模块
    ├── store.js            # IndexedDB:存/取/删导入的歌
    ├── player.js           # 音频播放 + Web Audio 应用内音量
    ├── wheel.js            # 点按轮盘:画圈角度检测 + 五键,输出抽象指令
    ├── screens.js          # 屏幕状态机(主菜单↔歌曲↔正在播放)
    ├── ui.js               # 按当前状态渲染屏幕
    └── colors.js           # 歌名→专属固定撞色映射 + 精选色盘
```

### 模块职责(各司其职,清晰接口通信)

| 模块 | 职责 | 依赖 | 对外接口(示意) |
|------|------|------|------------------|
| `store.js` | 数据持久化:存音频文件、读列表、删除 | IndexedDB | `addSong(file)`, `getAllSongs()`, `getSongBlob(id)`, `deleteSong(id)` |
| `player.js` | 声音:播/暂停/切歌/跳进度/音量,发播放状态事件 | Audio + Web Audio | `play(song)`, `pause()`, `next()`, `prev()`, `seek(t)`, `setVolume(v)`, `on('statechange', cb)` |
| `wheel.js` | 输入:把画圈/点击翻译成抽象指令 | 触摸事件 | 发出 `SCROLL_NEXT`/`SCROLL_PREV`/`SELECT`/`MENU`/`PREV`/`NEXT`/`PLAY_PAUSE` |
| `screens.js` | 导航状态机:当前屏幕 + 指令如何改变状态 | wheel 指令 + player 状态 | `dispatch(command)`, `getState()`, `on('change', cb)` |
| `ui.js` | 渲染:把状态画到屏幕 | screens 状态 | `render(state)` |
| `colors.js` | 由歌名稳定映射到色盘中一个撞色 | — | `colorForSong(title)` |
| `app.js` | 启动装配:wheel→screens→ui,player→ui | 以上全部 | — |

### 数据流
手指动 → `wheel` 出抽象指令 → `screens` 改状态 → `ui` 重绘;
播放时 `player` 状态变化 → `ui` 更新「正在播放」界面。
各模块解耦,可独立理解与测试。

## 5. 屏幕与导航

三个屏幕 + 一个导入入口。屏幕区在上半部,轮盘在下半部。

### 屏幕 1 · 主菜单(落地页)
- 两项:`歌曲` / `正在播放`,玫红高亮当前选中项。
- 画圈:上下移动高亮。中键:进入选中项。
- 右下角隐藏小入口(齿轮/＋)→ 导入。

### 屏幕 2 · 歌曲列表
- 画圈滚动列表(转得快滚得快)。
- 中键:播放选中歌 → 跳到「正在播放」。
- MENU(上键):返回主菜单。
- 空态:玫红图标 +「还没有歌,点这里导入」。

### 屏幕 3 · 正在播放(Now Playing)
- 背景:该歌的**专属固定撞色渐变**(见 §7)。
- 内容:歌名(柔白)+ 歌手(半透明白),进度条(玫红),已播/总时长。
- 左键 ⏮:开头 3 秒内→上一首,否则→回到开头。
- 右键 ⏭:下一首。
- 下键 ⏯:播放/暂停。
- 中心圆:循环切换画圈作用(音量 🔊 ↔ 进度 ⏩),屏幕上方小图标提示当前模式;长按中心圆 = 播放/暂停。
- 画圈:按当前模式调音量或调进度。
- MENU(上键):返回歌曲列表。

### 导航地图
```
主菜单 ──中键(歌曲)──▶ 歌曲列表 ──中键──▶ 正在播放
  │                      ▲                 │
  ├──中键(正在播放)──────┼─────────────────▶
  ▲                      │                 │
  └──────MENU────────────┴──────MENU────────┘
```
MENU 永远是"返回上一级",与真 iPod 一致。

## 6. 轮盘交互(wheel.js)

### 物理结构
```
        ┌─ MENU ─┐
  ⏮ ◀  │  ( ● )  │  ▶ ⏭
        └─  ⏯  ─┘
   └──── 外圈圆环(画圈)────┘
```

### 画圈检测
- 记录手指相对轮盘圆心的角度,计算角度增量。
- 顺时针累加 → `SCROLL_NEXT`,逆时针 → `SCROLL_PREV`。
- 每累计转过阈值(约 15°)发一格指令:转得快=滚得快。
- 触感:每格触发 `navigator.vibrate`(支持则用,默认开);咔哒音效设置开关,默认关。

### 五键
- 按下时按触点落区(上/下/左/右/中心圆)判定,发对应指令。
- 中心圆为独立按钮区,与外圈环分离,防误触。

### 画圈在不同屏幕的含义(由 screens 决定,wheel 只发 SCROLL)
| 屏幕 | 画圈 = |
|------|--------|
| 主菜单 / 歌曲列表 | 移动高亮 |
| 正在播放 | 按当前模式:音量 或 进度 |

### 进度↔音量切换
「正在播放」界面点击中心圆循环切换画圈作用(音量/进度),上方小图标提示;长按中心圆=播放/暂停。

### 边界
- 列表滚到顶/底:停住 + 轻微回弹,不循环穿越。
- 画圈时忽略中心圆区域滑动。

## 7. 视觉规范(高级黑 + 玫红点缀)

调性:深邃哑光黑为主,玫红仅作点睛(高亮/进度/关键反馈),克制使用以显高级。避免大面积红、避免高饱和荧光。

### 配色板
| 用途 | 颜色 |
|------|------|
| 外壳主体 | `#0A0A0C`(哑光近黑,微冷灰) |
| 外壳渐变 | `#1A1A1F → #0A0A0C`(微金属反光) |
| 屏幕背景 | `#101014` |
| 主文字 | `#EDEDF0`(柔白) |
| 次要文字 | `#6E6E78`(灰) |
| 玫红点缀 | `#E0476B` → 高光 `#FF6B8A` |
| 玫红辉光 | `rgba(224,71,107,0.35)` |

### 质感细节
- 轮盘:深黑圆盘 + 极细玫红描边环 + 内阴影(凹陷立体感);中心圆略亮,按下时玫红微亮 + 内陷动画。
- 屏幕:轻微内发光边框,圆角矩形;可选极淡扫描线/噪点纹理(复古数码屏)。
- 高亮项:玫红渐变填充 + 柔光辉光 + 文字转柔白,切换有顺滑滑动动画。
- 进度/音量条:细长,轨道深灰,已播放部分玫红渐变带微辉光,圆形小滑块。
- 字体:中文 `PingFang SC`,数字/英文系统几何无衬线,字距略宽。
- 动效:60fps 顺滑;菜单淡入滑动;按键细微缩放/发光反馈。

### 布局(竖屏)
```
┌─────────────┐
│   屏幕区      │  约 45%,圆角矩形
├─────────────┤
│   轮盘区      │  约 45%,居中大圆轮盘
└─────────────┘
   外壳留白 + 顶部状态条
```

### 「正在播放」专属撞色背景(colors.js)
- 不用纯随机 RGB(易出脏色)。预置一组**专门跟黑+玫红撞得好看**的精选色(深青绿、暗钴蓝、暗紫、墨绿、暗琥珀、酒红偏紫等)。
- 由歌名计算稳定哈希值映射到色盘中固定一色 → **每首歌一个专属固定色**(同歌每次都一样,成为其视觉身份)。
- 每块背景以该色→更深同色系做柔和渐变;歌名柔白、歌手半透明白叠加;必要时加暗角提升文字可读性。

## 8. PWA / 全屏

- `manifest.json`:`"display": "standalone"`,自定义图标、名称、`theme_color`/`background_color` 为深黑。
- `index.html` 加苹果专用标签:
  - `<meta name="apple-mobile-web-app-capable" content="yes">`
  - `<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">`
- 全屏前提:必须 Safari 分享 →「添加到主屏幕」,再从主屏幕图标打开,方无浏览器 UI(iOS 限制)。
- 首次在 Safari 内打开时,屏幕提示「添加到主屏幕,获得全屏体验」,已添加(standalone 模式)后不再提示。
- service worker 缓存应用外壳,离线可打开。

## 9. 错误处理与边界

| 情况 | 处理 |
|------|------|
| 无导入歌曲 | 空态引导导入 |
| 不支持的格式 | 提示,仅接受浏览器可放格式(mp3/m4a/aac/wav;flac 视 iOS 版本) |
| 文件过大/存储满 | 捕获 IndexedDB 写入失败,提示清理 |
| 解码/播放失败 | 提示并自动跳下一首,不卡死 |
| 列表到顶/底 | 停住 + 回弹 |
| 播到最后一首结束 | 停止(MVP 不循环) |
| 存储被 iOS 清理 | 显示空态,重新导入即可 |
| 未加主屏幕 | 顶部提示引导;加后不再提示 |
| 锁屏/后台 | 尽量保持播放 + Media Session API 显示锁屏信息;不过度承诺 iOS PWA 后台限制 |

## 10. 测试策略

务实为主(纯前端小 app):
- **逻辑单元测试**:`wheel.js` 角度→指令换算、`screens.js` 状态机跳转、`store.js` 存取、`colors.js` 歌名→颜色稳定性。轻量断言(Node 或浏览器内),不引重型框架。
- **手动验收清单**:在 iPhone 真机逐项走查(导入→列表→播放→切歌→画圈滚动→音量/进度切换→MENU 返回→关掉重开歌还在→添加主屏幕全屏)。
- 不追求覆盖率;UI 动效/视觉靠真机肉眼验收,不写脆弱 UI 自动化。

## 11. 已知限制(诚实声明)

- iOS Safari 网页无法改系统音量 → 用 Web Audio 应用内音量替代。
- iOS Safari 对网页存储有容量上限,且长期不用可能被系统回收(加主屏幕、常用可降低概率;被清后重新导入)。
- iOS PWA 后台/锁屏播放控制受限,只做到平台允许的程度。
- 全屏需用户手动「添加到主屏幕」一次。
